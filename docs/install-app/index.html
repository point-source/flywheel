Install the swarmflow App

swarmflow's pipeline runs as a GitHub App you control. This page creates one with the exact permissions the workflows need — no manual checkbox-clicking on GitHub's "New GitHub App" form.

1. Create the App

Pick where the App should live. You can install it on multiple repos later.

GitHub will land you on a partially-prefilled "New GitHub App" form. Set a name, leave the rest as-is, click Create GitHub App.

2. After creation

  1. Copy the numeric App ID from the top of the App's settings page.
  2. Scroll to Private keys → click Generate a private key. A .pem file downloads.
  3. Click Install App in the left sidebar and install it on each repo where you want swarmflow to run. Choose "Only select repositories" and pick them explicitly.
  4. On every adopter repo, set two Actions secrets: 
    gh secret set APP_ID --repo <owner>/<repo> --body "<app-id>"
gh secret set APP_PRIVATE_KEY --repo <owner>/<repo> < path/to/key.pem

Permissions this App requests

PermissionLevelWhy
Contentsread & writepush, tag, write CHANGELOG
Pull requestsread & writeopen, edit, merge PRs
Checksread & writepost status / quality results
Actionsread & writedispatch build / publish / quality workflows
Metadatareadrequired by GitHub for every App

The App is not granted Admin. It bypasses specific rulesets when configured to (see the swarmflow RULESETS.md).
